RHN Proxy Server: Installation and Configuration Guide; Version 1.7.0

Chapter 2. Software Requirements

The following software requirements should be met by installing Red Hat Linux 7.2 or Red Hat Linux Advanced Server 2.1 and subscribing the system to the RHN Proxy Server channel via Spacewalk.

The current version of the RHN Proxy Server utilizes the services of the Apache Web Server and Squid, an HTTP proxy caching server:

  • Apache Web Server — The version of Apache from the Red Hat Linux/Advanced Server distribution along with all the Errata Updates should be sufficient. It should be configured to listen on port 80.

  • HTTP proxy caching server — Squid is recommended and has been tested to work with RHN Proxy Server. It is part of the Red Hat Linux/Advanced Server distribution. However, any HTTP/1.1 compliant proxy caching server should suffice. It should be configured to listen on port 8080.

To install and use the RHN Proxy Server solution, Red Hat Linux 7.2 or Red Hat Linux Advanced Server 2.1 and the following RPM packages are required in addition to the Apache Web Server and Squid where indicated.

Warning
 

Although your system may already have python installed, mod_python (the Apache Python binding mechanism) requires that the non-threaded version of Python be installed. Do not use the stock python and mod_python packages from Red Hat Linux 7.2 or Red Hat Linux Advanced Server 2.1. Install the versions distributed with the RHN Proxy Server RPM packages.
